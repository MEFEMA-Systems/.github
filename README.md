O conteúdo abaixo foi elaborado com base nas informações públicas disponíveis em https://mefemasys.co.mz e em boas práticas de governança de código. Revise e ajuste conforme necessário antes de publicar.

## Visão geral

A MEFEMA Systems fornece soluções tecnológicas e serviços digitais direcionados a empresas e instituições, com foco em transformação digital, desenvolvimento de software e suporte técnico. Este repositório centraliza arquivos que padronizam a colaboração, revisão e publicação de código em todos os projetos da organização.

## Objetivos deste repositório (.github)

- Padronizar templates de issues e pull requests.
- Centralizar políticas organizacionais (CONTRIBUTING, CODE_OF_CONDUCT, SECURITY).
- Fornecer orientações para colaboradores e processos de revisão.
- Compartilhar workflows e configurações reutilizáveis (GitHub Actions) quando aplicável.

## Como contribuir

1. Identifique o repositório do projeto dentro da organização.
2. Leia CONTRIBUTING.md e CODE_OF_CONDUCT.md antes de contribuir.
3. Abra uma issue para discutir mudanças significativas quando necessário.
4. Faça fork (ou branch) do repositório, implemente a alteração em uma branch com nome descritivo (ex.: feat/nome-da-feature).
5. Submeta um pull request seguindo o template, descrevendo objetivo, mudanças, testes e impactos.

## Boas práticas de commits e PRs

- Use mensagens de commit curtas e descritivas.
- Separe alterações em commits lógicos e atômicos.
- No PR inclua descrição, checklist e instruções de teste.
- Adicione reviewers e labels conforme apropriado.

## Segurança

Se encontrar vulnerabilidades, reporte de forma responsável conforme descrito em SECURITY.md. Forneça passos para reproduzir e evite divulgar publicamente até correção.
